Decisão da Arquitetura Utilizada

Para este projeto, optei por uma arquitetura modular, dividindo as funcionalidades em diferentes módulos para melhor organização e escalabilidade. Utilizei uma abordagem orientada a objetos para encapsular as diferentes funcionalidades, como extração de texto de PDFs, transcrição de vídeos, interação com APIs de IA generativa, e indexação de dados.

Lista de Bibliotecas de Terceiros Utilizadas

  • - PaddleOCR: Utilizada para extração de texto de imagens, especificamente de PDFs.
  • - moviepy: Usada para extrair áudio de arquivos de vídeo MP4.
  • - SpeechRecognition: Empregada para transcrição de áudio em texto.
  • - elasticsearch: Utilizada para indexação de dados, permitindo uma busca eficiente por palavras-chave e frases relevantes.
  • - openai: Utilizada para interação com o modelo GPT-4, possibilitando a geração dinâmica de conteúdo adaptativo em resposta às solicitações dos usuários.
  • - google-generativeai: Utilizada para interação com o modelo Gemini, proporcionando a geração de conteúdo adaptativo.

O Que Você Melhoraria Se Tivesse Mais Tempo

Se tivesse mais tempo, algumas melhorias que eu consideraria incluir seriam:

  • - Testes Unitários e TDD: Implementaria testes unitários para garantir a robustez e a qualidade do código, seguindo a metodologia TDD (Desenvolvimento Orientado a Testes).
  • - Melhoria na Indexação: Substituiria a indexação atual por uma integração com o ElasticSearch para uma indexação mais eficiente e escalável, proporcionando uma busca mais rápida e relevante.
  • - Implementação Completa da Indexação no Código: Integraria totalmente a funcionalidade de indexação no código-fonte, automatizando o processo de indexação de dados.
  • - Necessito entender as regras de negócio, para ter ciência dos gargalos para pensar na melhor lógica de implemetação.
  • Neste projeto, busquei apresentar um trabalho bem executado, dadas as minhas limitações de tempo. Consegui completá-lo em menos de 10 horas de trabalho direto, e no dia a dia, estou comprometido em atender às necessidades reais e superar as expectativas ao máximo.

Requisitos Obrigatórios que Não Foram Entregues

Estes requisitos não foram completamente entregues devido ao foco principal no desenvolvimento das funcionalidades de indexação de dados e interação com modelos de IA generativa.

  • - Implementação de um prompt de aprendizagem adaptativa interativo.
  • - Análise das dificuldades dos usuários e geração de conteúdos dinâmicos adaptados.
  • - Integração de diferentes componentes do sistema de forma coesa.
